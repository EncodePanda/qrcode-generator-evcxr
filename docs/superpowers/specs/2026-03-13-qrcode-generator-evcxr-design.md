# qrcode-generator-evcxr Design Spec

## Overview

A thin Rust library wrapping `qrcode-generator` to display QR codes as SVG images in Jupyter notebooks via the evcxr kernel.

## Public API

### Functions

```rust
/// Display a QR code in Jupyter using all default settings.
pub fn draw_qrcode(input: impl QrCodeable) { ... }

/// Display a QR code in Jupyter with custom configuration.
pub fn draw_qrcode_with_config(input: impl QrCodeable, config: QrConfig) { ... }
```

`draw_qrcode` delegates to `draw_qrcode_with_config` with `QrConfig::default()`.

### QrConfig

```rust
pub struct QrConfig {
    pub ec_level: QrCodeEcc,  // Error correction level (re-exported from qrcode-generator)
    pub size: usize,          // SVG pixel size
    pub margin: usize,        // Quiet zone in modules
}

impl Default for QrConfig {
    fn default() -> Self {
        QrConfig {
            ec_level: QrCodeEcc::Medium,
            size: 256,
            margin: 4,
        }
    }
}
```

### QrCodeable Trait

```rust
pub trait QrCodeable {
    fn to_matrix(&self, ec_level: QrCodeEcc) -> Vec<Vec<bool>>;
}
```

Implementations:

- **`str`** and **`String`** — calls `qrcode-generator`'s `to_matrix_from_str` with the given EC level.
- **`Vec<Vec<bool>>`** — returns a clone of itself, ignores `ec_level` (the matrix is already generated).
- **`&T` where `T: QrCodeable + ?Sized`** — forwarding impl so `&str`, `&String`, `&&str`, etc. all work.

Note: A blanket `impl<T: AsRef<str>>` would be cleaner but conflicts with the `Vec<Vec<bool>>` impl in Rust's type system.

### Re-exports

`QrCodeEcc` is re-exported from `qrcode-generator` so users don't need to depend on it directly.

## Internal Flow

1. `draw_qrcode(input)` calls `draw_qrcode_with_config(input, QrConfig::default())`
2. `draw_qrcode_with_config` calls `input.to_matrix(config.ec_level)` to get `Vec<Vec<bool>>`
3. An internal `render_svg(matrix, size, margin)` function produces an SVG string from the bool matrix. This is necessary because `qrcode-generator`'s SVG functions go directly from string input to SVG — there is no function to convert a matrix to SVG, and no margin parameter. The renderer draws a grid of filled rectangles for `true` cells on a white background, scaled to the configured size with the configured quiet zone margin.
4. Prints via evcxr display protocol:
   ```
   EVCXR_BEGIN_CONTENT image/svg+xml
   <svg>...</svg>
   EVCXR_END_CONTENT
   ```

## Dependencies

- `qrcode-generator` — QR code matrix generation (via `to_matrix` / `to_matrix_from_str`)

No other dependencies required. SVG is generated by a simple internal renderer (rectangles in a grid). No `base64` or `image` crate needed.

## Error Handling

Functions panic with descriptive messages if QR code generation fails. This is consistent with the `petgraph-evcxr` pattern — Jupyter notebook contexts don't benefit from `Result` return types.

## Example Usage in Jupyter

```rust
:dep qrcode-generator-evcxr = "*"
use qrcode_generator_evcxr::{draw_qrcode, draw_qrcode_with_config, QrConfig, QrCodeEcc};

// Simple — all defaults
draw_qrcode("https://example.com");

// With a String
let url = String::from("https://example.com");
draw_qrcode(&url);

// With custom config
let config = QrConfig { ec_level: QrCodeEcc::High, ..Default::default() };
draw_qrcode_with_config("https://example.com", config);

// With a pre-built matrix
let matrix: Vec<Vec<bool>> = /* from qrcode-generator */;
draw_qrcode(matrix);
```

## Project Structure

```
qrcode-generator-evcxr/
  Cargo.toml
  src/
    lib.rs        # All public API, trait, config, and internal rendering
```

Single-file library — the codebase is small enough that splitting into modules would be unnecessary.
